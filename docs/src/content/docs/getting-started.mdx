# Prerequisites

You will need to have a **Starlight** website set up.
If you don't have one yet, you can follow the ["Getting Started"](https://starlight.astro.build/getting-started) guide in the Starlight docs to create one.

# Installation

import { Steps, Tabs, TabItem } from '@astrojs/starlight/components'

<Steps>

1. ` @six-tech/starlight-theme-six` is a Starlight [plugin](https://starlight.astro.build/reference/plugins/). Install it by running the following command in your terminal:

   <Tabs syncKey="pkg">

   <TabItem label="npm">

   ```sh
   npm install @six-tech/starlight-theme-six @fontsource/inter @fontsource/jetbrains-mono
   ```

   </TabItem>

   <TabItem label="pnpm">

   ```sh
   pnpm add @six-tech/starlight-theme-six @fontsource/inter @fontsource/jetbrains-mono
   ```

   </TabItem>

   <TabItem label="Yarn">

   ```sh
   yarn add @six-tech/starlight-theme-six @fontsource/inter @fontsource/jetbrains-mono
   ```

   </TabItem>

   </Tabs>

2. Configure the plugin in your Starlight [configuration](https://starlight.astro.build/reference/configuration/#plugins) in the `astro.config.mjs` file.

   :::tip[Automatic Defaults]
   The Six theme automatically provides sensible defaults for:
   - **Logo**: Six theme logos (dark/light variants)
   - **Title**: "Starlight Six" 
   - **Fonts**: Inter and JetBrains Mono (all weights included)
   - **Last Updated**: Enabled by default
   - **Table of Contents**: Shows H1-H5 headings
   
   You can override any of these by providing your own values in the Starlight configuration.
   
   **Note:** Favicons are not automatically configured to avoid base path conflicts. Use the `getFavIcons()` utility for manual setup.
   :::

   ```diff lang="js"
   // astro.config.mjs
   import starlight from '@astrojs/starlight'
   import { defineConfig } from 'astro/config'
   +import starlightThemeSix from '@six-tech/starlight-theme-six'

   export default defineConfig({
     integrations: [
       starlight({
   +      plugins: [
   +        starlightThemeSix({
   +          navLinks: [{ // optional
   +            label: 'Docs',
   +            link: '/getting-started',
   +          }],
   +          footerText: //optional
   +            'Built & designed by [shadcn](https://twitter.com/shadcn). Ported to Astro Starlight by [AdriÃ¡n UB](https://github.com/adrian-ub). The source code is available on [GitHub](https://github.com/six-tech/starlight-theme-six).'
   +       })
   +      ],
         title: 'My Docs',
       }),
     ],
   })
   ```

3. [Start the development server](https://starlight.astro.build/getting-started/#start-the-development-server) to preview the plugin in action.

</Steps>

## Minimal Configuration

With the automatic defaults, you can get started with just the plugin configuration:

```js title="astro.config.mjs"
import starlight from '@astrojs/starlight'
import { defineConfig } from 'astro/config'
import starlightThemeSix from '@six-tech/starlight-theme-six'

export default defineConfig({
  integrations: [
    starlight({
      plugins: [
        starlightThemeSix({
          // Theme-specific options (all optional)
          navLinks: [
            { label: 'Docs', link: '/getting-started' },
          ],
        })
      ],
      // That's it! The theme provides defaults for logo, title, favicons, etc.
    }),
  ],
})
```

## Overriding Defaults

You can override any default by providing your own values:

```js title="astro.config.mjs"
import { getFavIcons } from '@six-tech/starlight-theme-six/utils/favicons'

export default defineConfig({
  integrations: [
    starlight({
      plugins: [starlightThemeSix({ /* theme config */ })],
      
      // Override defaults
      title: 'My Custom Title',
      logo: {
        src: './src/assets/my-logo.svg',
      },
      lastUpdated: false, // Disable last updated
      tableOfContents: {
        minHeadingLevel: 2, // Start from H2
        maxHeadingLevel: 4, // End at H4
      },
      head: [
        // Add favicon configuration manually
        ...getFavIcons({ basePath: '/my-project/' }), // For subdirectory deployment
        // Or for root deployment: ...getFavIcons()
      ],

    }),
  ],
})
```
